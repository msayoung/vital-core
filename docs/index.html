vital-core

A serverless, spec-driven quality scanner that checks accessibility, performance, plain language, and sustainability for U.S. government websites — powered by GitHub Actions.

🔓 Open Source ⚡ GitHub Actions ♿ Section 508 🌱 Sustainability 📊 WCAG 2.2

Build Phases

Data Schema & Profile Ingestion

Zod-validated YAML profiles define scan targets, glob path filters, priority URLs, and execution ceilings.

Target Discovery & Path Filter Matrix

Key Features

🗺️

Glob-Filtered Sitemap Parsing

Parses remote sitemap.xml and cross-references entries with picomatch wildcard rules to isolate targeted paths.

🚦

Priority Seed Injection

High-value URLs are forced to the front of the execution queue, guaranteeing they are audited regardless of crawl depth limits.

♿

Section 508 / WCAG Auditing

Hydrated DOM axe-core evaluations map violations directly to Section 508 Functional Performance Criteria.

📝

Plain Language Scoring

Flesch-Kincaid Reading Ease scores enforce compliance with the Plain Writing Act of 2010 on every scanned page.

🌱

Carbon & Sustainability Metrics

Estimates CO₂ per visit using Sustainable Web Design model calculations derived from live network transfer data.

⚙️

100% Serverless via GitHub Actions

No cloud infrastructure costs. Every scan runs on GitHub-hosted runners on a cron schedule, fully automated.

🛡️

Single-Hit Bandwidth Protection

Pages are fetched exactly once. Raw HTML is cached locally, and all offline audits run against the snapshot to protect federal bandwidth.

🎫

Automated Accessibility Tickets

Violations are transformed into ACCESSIBILITY.md-formatted Markdown tickets, ready to paste into GitHub Issues or Jira.

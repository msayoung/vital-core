## Project Purpose

vital-core is an open-source website quality engine. It crawls government websites using Playwright and Chromium, runs accessibility, performance, sustainability, and other quality checks, and generates static HTML reports published to GitHub Pages.

Current assessment engines include:

- axe-core
- Siteimprove Alfa
- Lighthouse

The project objective is to identify the most important website quality problems and present them to the people best positioned to address them. Prioritization should be based on user impact, accessibility risk, sustainability, public benefit, and the scale of affected pages.

## Testing Requirements

Required before merge:

```bash
npm run test:unit
npm run test:e2e
```

Requirements:

- All unit tests must pass.
- Existing tests must not be broken.
- New code must not introduce security regressions.

## Quality Requirements

- Never commit `.env` files, authentication tokens, or secrets.
- Local and Hugging Face data/state paths must remain gitignored.
- Targets marked `hf_only: true` are filtered out in `scan.yml` CI.
- CSS changes belong in the CSS constant in `src/report-html.js`, not in generated files under `docs/`.
- The project must run on macOS, Linux, and GitHub Actions using `ubuntu-latest`.
- Generated reports should support review against WCAG 2.2 AA.
- AI usage and its role in findings or recommendations must be disclosed.

## Accessibility Severity Taxonomy

Accessibility findings must use the axe-core severity taxonomy:

- Critical
- Serious
- Moderate
- Minor

Do not replace these values with High, Medium, or Low.

## Default Reporting Policy

By default, VITAL displays:

- WCAG A and AA findings that are Critical or Serious.
- Moderate and Minor findings affecting 10 or more pages.

Best Practice findings are hidden by default. A "Show everything" control reveals all findings.

## Architecture Conventions

- Engine modules belong in `src/engines/`.
- New engines must be wired into `src/scan.js` and `src/aggregate.js`.
- Report UI CSS changes belong in `src/report-html.js`.
- Deployment uses GitHub Pages from the `docs/` artifact generated by `aggregate.js`.

## Branching and Review

- Feature branches originate from `main`.
- Pull requests target `main`.
- Do not force-push to `main`.
- At least one focused reviewer must approve before merge.

## Security Requirements

- Never commit `.env`.
- Never commit authentication tokens.
- Keep local and Hugging Face state paths gitignored.
- Keep `hf_only: true` targets out of standard CI workflows.
